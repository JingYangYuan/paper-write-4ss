# 中文学术写作系统：标准与迭代

本系统基于《社会学研究》等权威期刊实际发表论文的结构特征、论证模式和语言规范提炼而成。所有标准均从已发表论文中归纳，每一条都可以在参考文献论文中找到对应的实例。

---

## 零、研究范式选择（使用前必读）

在开始论文写作之前，必须先通过以下三个问题确定写作范式。Skill 加载后，应使用 AskUserQuestion 工具向用户提问：

### 问题一：目标期刊风格

用户的目标期刊属于哪种风格？

- **社会学研究风格**：《社会学研究》《社会》《中国社会科学》《社会学评论》《青年研究》等社会学期刊
- **管理世界风格**：《管理世界》《公共管理学报》《中国行政管理》《公共行政评论》等管理/公共管理期刊
- **尚未确定**：根据后续问题判断

### 问题二：研究方法

用户论文使用什么研究方法？

- **量化实证**：调查数据、统计模型、计量分析 → 倾向社会学研究范式
- **质性田野/深度访谈/民族志**：需要进一步判断是偏向学术脉络对话还是机制建构
- **案例研究（单/多/纵向/嵌入式）**：倾向管理世界案例研究范式
- **理论分析/概念研究**：倾向社会学研究范式

### 问题三：预期字数

论文的预期字数（不含参考文献）？

- **1.5 万字以下**：短篇论文，两种范式均可
- **1.5-2.5 万字**：标准期刊论文篇幅
- **2.5 万字以上**：大型研究，需注意各部分比例均衡

### 范式判定

根据三个问题的回答综合判定：

- 社会学期刊 + 量化/理论方法 → **社会学研究范式（定量/理论路径）**
- 社会学期刊 + 质性田野/民族志 → **社会学研究范式（质性路径）**
- 管理世界期刊 + 案例研究 → **管理世界案例研究范式**
- 跨期刊投稿 + 案例研究 + 关注实践问题 → **管理世界案例研究范式**
- 跨期刊投稿 + 非案例方法 + 关注学术脉络 → **社会学研究范式**

范式判定后应明确告知用户：选定的范式、该范式下各章节的核心要求、以及相应的范文和提示词入口。

**下文各章节的硬性标准以社会学研究范式为基准**；管理世界范式的差异化要点在各章节中以"管理世界范式"标记单独说明。详细的范式对比、决策流程和转换指南另见 [resources/paradigm_selector.md](resources/paradigm_selector.md)。

## 一、论文七要素标准

一篇完整的学术论文由七大要素构成，缺一不可：

| # | 要素 | 核心功能 | 篇幅参考 |
|---|------|---------|---------|
| 1 | 标题 + 摘要 + 关键词 | 凝练全文精华，供检索和第一判断 | 标题 ≤ 25 字；摘要 300-500 字 |
| 2 | 引言（问题的提出） | 论证研究合法性 | 约占全文 10-15% |
| 3 | 文献综述 / 理论框架 | 定位知识脉络，建立分析框架 | 约占全文 15-20% |
| 4 | 研究方法与数据 | 叙事性呈现研究过程 | 约占全文 8-12% |
| 5 | 经验分析 / 研究发现 | 以逻辑组织材料，呈现发现 | 约占全文 30-40% |
| 6 | 理论对话（可嵌入经验分析或结论） | 回到知识脉络，提炼理论贡献 | 约占全文 10-15% |
| 7 | 结论与讨论 | 总结全文，延展意义 | 约占全文 10-15% |

## 三、全文质量检查清单

论文提交前，按以下顺序逐项检查：

### 第一轮：结构完整性
- [ ] 七大要素齐全（标题、摘要关键词、引言、文献综述、方法、经验分析、结论与讨论）
- [ ] 各部分篇幅比例合理（经验分析 30-40%，其余部分均衡）
- [ ] 各级标题逻辑连贯（只看目录即可判断论文骨架清晰）

### 第二轮：论证严密性
- [ ] 引言中有转折词（然而/但是/不过）+ 研究缺口 + 贡献陈述
- [ ] 文献综述有分析框架 + 各分节交汇
- [ ] 经验分析各分节递进，有"三层递进"
- [ ] 理论对话回到引言的脉络，有比较性讨论
- [ ] 结尾六要素覆盖，不以限制结尾

### 第三轮：材料与引文
- [ ] 引文呈现形式多样化（≥ 3 种）
- [ ] 引文后有思考痕迹（≥ 1 句分析/每处）
- [ ] 文献引用前后密度均衡（前后半差异 ≤ 2 倍）
- [ ] 中文 + 英文混合引用

### 第四轮：语言规范
- [ ] 无"一文中"、"该文"等非学术口语
- [ ] 无"以...为例"等套路化表达（标题中尤其是）
- [ ] 无"本文首次..."等虚假创新宣称（除确属创新）
- [ ] 无未定义缩写
- [ ] 无正文中嵌入完整论文标题（应用作者-年份格式）
- [ ] 引号全部为中文引号 `""`，无英文直引号 `""`

## 四、写作约束

以下约束适用于本 Skill 生成或指导的所有学术文本：

1. **禁止在正文行文中嵌入完整论文标题**。使用作者-年份引用格式（如"卢福营和张兆曙（2006）"），读者可通过参考文献列表找到完整标题。

2. **禁止使用"一文中""该文"等非学术口语表述**。来自英文"in this paper/article"的直译惯性在中文语境中不合学术规范。应改写为"该研究"或直接陈述研究发现。

3. **禁止使用"不是……而是……"句式及其所有变体**。以直接陈述替代先否定后肯定的对立结构。此约束适用于本 Skill 生成或润色的所有文本（包括用户论文）。

  **禁用变体（非穷举）**：
  - `不是 A，而是 B` → 直接陈述 B 即可
  - `并非 A，而是 B` → 同上
  - `不在于 A，而在于 B` → 改写为 `关键在于 B`，`A 并非要害`
  - `并不意味着 A，而是 B` → 改写为 `本质上是 B`
  - `不只是 A，更是 B` → 改写为 `既是 A，更是 B` 或直接陈述 B
  - `不……，而……` 结构 → 均需改写

  **改写示例**：
  - ❌ `劳动是否可见，并不取决于它是否重要，而取决于组织如何定义`
  - ✅ `劳动是否可见，取决于组织如何定义。这与劳动本身的重要性无关`
  - ❌ `并非研究者能力不足，而是基础设施缺失`
  - ✅ `根源在于基础设施缺失，研究者能力不足只是表象`

4. **正文禁止使用破折号（——）**。包括但不限于：插入语、解释说明、转折强调、列举引述等所有场景。若句子结构依赖——才能成立，说明句子本身需要重写。参考文献条目中的论文标题（如"主标题——副标题"）除外。

5. **禁止使用脚本机械替换标点符号**。所有文本修改必须由 AI 自然重写句子，不得用正则或字符串替换将 `——` 替换为 `：`/`，`/`。` 等标点。

6. **禁止不必要地使用引号**（必要最少原则）。引号仅用于以下场景：(a) 直接引述原文，(b) 首次提出的特殊概念/术语，(c) 反讽或非常规用法。不得将引号用于强调、装饰普通词语、或包装非直接引述的内容。**能不使用引号的地方，一律不使用**。学术写作中，大部分引号都是不必要的——直接陈述即可。

7. **正文引号必须使用中文引号 `""`（U+201C + U+201D）**。禁止使用英文直引号 `""`（U+0022）或半角引号。左引号为 `"`，右引号为 `"`，不得混用。

### 润色循环工作流

所有文本修改必须遵循**扫描 → 指令 → 重写 → 再扫描**循环，不允许脚本机械替换：

```
1. 运行 writing_scanner.py --instructions 生成改写指令 MD
2. AI 阅读指令，逐条自然重写句子（不是替换标点）
3. 运行 writing_scanner.py --scan 验证
4. 若仍有检出 → 回到步骤 1
5. 循环至 🔴高危 归零
```

## 五、范文与提示词系统

本 Skill 为论文的五大核心章节提供了完整的范文和可复用的提示词模板。每篇范文均带有逐段结构标注（对应上文的硬性标准），便于对照学习和模仿写作。

### 使用方式

1. **查阅范文**：找到对应章节的范文文件，阅读带标注的范文文本，理解每个段落的功能定位
2. **复制提示词**：根据论文类型选择对应的提示词模板，填入论文的具体信息
3. **生成或修改文本**：将提示词作为 AI 写作指令，生成或修改对应章节

### 范文标注体系

所有范文使用以下标注对应 SKILL.md 的硬性标准：

- 引言：[I1] 现象引入 [I2] 转折与不足 [I3] 缺口具体化 [I4] 切入点 [I5] 贡献陈述 [I6] 自我陈述 [I7] 不以结构预告结尾
- 文献综述：[L1] 分析框架 [L2] 分节功能 [L4] 交汇段落 [L5] 引文多样性 [L6] 引文后分析

- 结论：[C1] 总结全文 [C2] 纵横延展 [C3] 视角独特性 [C4] 遗留问题 [C5] 研究限制 [C6] 未来展望
